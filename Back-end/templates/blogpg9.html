A deep look into comparison of CSV and JSON files.

by Guy Hawkins

An overview and structure JSON.

JSON is a data exchange format that stands for JavaScript Object Notation, files are saved in the extension .json. Commonly known as a light-weight data interchange format type and favored for its human readability and nesting features, it is often used in conjunction with APIs and data configuration. It is used as the syntax for storing and exchanging data. It is referred to as the subset of JavaScript and the ECMA script. The structure of JSON is based on the name/value pair in which data is represented. Curly braces to hold objects and each name followed by the colon, and the comma separates the pairs, the square brackets to hold arrays and values. JSON is being supported by all the programming languages like Java, Net (C#), PHP, Perl, Python, Ruby, etc.

Pros and Cons of JSON

JSON is quite a good file format and mostly preferred by data scientists and software developers for the following reasons:

(i) JSON is fairly easy to read and write. It is more succinct then XML and works very well with Javascript and Actionscript. However it is not structured and validated the way XML is. This makes keeping generator and consumer synchronized an exercise in discipline and can fail in nasty ways at run time if you don't.

(ii) JSON's structure directly corresponds to basic data types of most of script languages such as array and map. It makes JSON API for script languages very simple e.g. just JSON.parse() which takes JSON string and returns a map or an array. XML structure doesn't have such obvious correspondence, so you need to use more complicated API such as DOM.

(iii) JSON has distinction between string, number and boolean e.g. {"count": "1"} vs {"count": 1}. XML document itself doesn't have the distinction. The distinction saves a bit of programmers work to convert to appropriate type manually. You can define data type of values in XML with XML Schema, but it's complicated and not always available.

However, there are some downsides of JSON files as well:

It can be quite dangerous if used with untrusted services or untrusted browsers, because a JSON service returns a JSON response wrapped in a function call, which will be executed by the browser if it will be used with untrusted browser, because it can be hacked, this makes the hosting Web Application Vulnerable to a variety of attacks.


An overview and structure of CSV.

CSV is an abbreviation for Comma Separated Value, a data storage format with the extension .csv. CSV files store data values (plain text) in a list format separated by commas. Notably, CSV files tend to be smaller in size and can be opened in text editors. CSV is noted for its small file size and simplicity. Likewise, because of its simplicity, CSV can be used by virtually anyone who is tasked with examining simple data in spreadsheets and tables.

Pros and Cons of CSV

CSV is mostly used by data professionals and software developers due to the numerous benefits it offers. Some are:
(i) CSV is human readable and easy to edit manually.
(ii) CSV is simple to implement and parse.
(iii) It is processed by almost all existing applications.
(iv) It is faster to handle.
(v) It provides a straightforward information schema.

Despite the numerous benefits and flexibility CSV offers, many people still opt for other alternatives due to the following:

(i) CSV has a compact file size.
(ii) Errors when displaying hierarchical data.
(iii) Not easily scalable and difficult to integrate.
(iv) Convenient for small datasets.

Conclusion

Both CSV and JSON files are good to work with based on the project you have at hand, as they have different range of flexibility and amount of data they can handle. If you want a file easy to scale and integrate with APIs easily, JSON is your best option. If your dataset is small and you want something simple to implement, go for CSV.
