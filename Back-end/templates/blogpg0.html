JSON data structure, what is it made of?

by Jacob Jones

Understanding the structure of JSON is a challenge for most people who work with data. Having a good knowledge of the structure will help you work better with JSON files. Here`s the breakdown of the JSON Data Structure: JSON defines only 2 data structures: objects and arrays. An object is a set of name-value pairs, and an array is a list of values. JSON defines seven value types: string, number, object, array, true, false, and null.

The Data types

A JSON object contains zero, one, or more key-value pairs, also called properties. The object is surrounded by curly braces {}. Every key-value pair is separated by a comma. The order of the key-value pair is irrelevant. A key-value pair consists of a key and a value, separated by a colon (:). The key is a string, which identifies the key-value pair. The value can be any of the following data types:

  • string- surrounded by quotation marks (“ ”)
  • number- positive or negative integers or floating point numbers
  • array- a collection of one or more JSON objects
  • object- a key, value pair enclosed in curly braces
  • boolean- (true or false)
  • empty- indicates the absence of data for a key value pair, represented as “null” with no quotes.
Structure, Syntax and Usage

JSON`s simplicity makes it easy to work with. It’s easy to write, read and translate between the data structures used by most languages. Let’s look at what makes up a JSON object, the data types that JSON supports, and other specifics with the syntax of this popular data format. Chances are you’ve seen JSON as you’ve looked through data or API documentation. The criteria for valid JSON is rather elementary, though it can be used to describe complex data. The structure of a JSON object is as follows:

(i) Curly braces {} hold objects

(ii) The data are in key, value pairs

(iii) Square brackets [] hold arrays

(iv) Each data element is enclosed with quotes if it‘s a character, or without quotes if it is a numeric value

(v) Commas are used to separate pieces of data

Source: Nylas.com; The Complete Guide to Working with JSON: Ben Lloyd Pearson
